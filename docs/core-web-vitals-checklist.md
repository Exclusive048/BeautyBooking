# Core Web Vitals: чек-лист

## LCP (Largest Contentful Paint)
- Использовать `next/image` для крупных изображений и указать размеры.
- Не блокировать LCP тяжелыми шрифтами: `font-display: swap`, preload только нужных начертаний.
- Не грузить тяжелые виджеты до первого рендера (динамический импорт, `ssr: false` при необходимости).
- Сократить размер hero-изображений, использовать `priority` только для одного ключевого.

## CLS (Cumulative Layout Shift)
- Всегда задавать размеры изображений и медиа.
- Резервировать высоту для блоков с асинхронной загрузкой.
- Не вставлять новые элементы выше уже отрендеренных без анимации/резервирования места.

## INP (Interaction to Next Paint)
- Разносить тяжелые обработчики по `requestIdleCallback`/`setTimeout`.
- Дробить большие списки, использовать виртуализацию при необходимости.
- Минимизировать синхронные вычисления в обработчиках кликов.

## Кэширование и сеть
- Включить кэширование для статических ресурсов.
- Использовать CDN для изображений/медиа.
- Минимизировать количество запросов на первом экране.

## Бандл и код-сплит
- Динамически импортировать тяжелые графики/виджеты.
- Удалять неиспользуемые зависимости и модули.

## Шрифты
- Ограничить число шрифтов и начертаний.
- Предзагружать только критичные шрифты.

## Проверка
- Lighthouse: LCP, CLS, INP в зеленой зоне.
- WebPageTest / Chrome UX Report: мониторить изменения после релизов.
